# Invocation Template

Use `project-collaboration-operating-system` for this project.

Mode: bootstrap / audit / improve / retrospective

Goal:
[describe what you want]

Route hint:

- `bootstrap`: no stable collaboration context.
- `audit`: existing docs or repo need diagnosis.
- `improve`: known collaboration gap needs the smallest safe fix.
- `retrospective`: real miss or repeated failure needs Failure Review.

Constraints:

- Do not modify files unless explicitly asked.
- Preserve existing project conventions and uncommitted user work.
- Classify failures before proposing document or workflow changes.
- Do not expand into product strategy, PRD, roadmap, market research, or full lifecycle planning.

Output:

- Selected mode and why.
- Current collaboration reality.
- Context and authority map.
- Active scope and deferred capabilities.
- Diagnosed need or failure type.
- Smallest justified system intervention.
- Verification or follow-up checks.
- Durable learning to preserve, if any.
